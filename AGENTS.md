# Antigravity Global Standards & Communication Rules

이 문서는 Antigravity AI 에이전트가 본 프로젝트를 수행함에 있어 준수해야 할 최상위 원칙을 정의합니다.

## 1. 커뮤니케이션 및 승인 원칙
- **No Emojis & No Fluff**: 전문적이고 명확한 텍스트로만 소통한다.
- **선 답변 후 작업**: 질문이나 설명 요청 시, 파일 수정이나 명령 실행을 금지하고 상세한 텍스트 답변을 최우선으로 제공한다.
- **수정 승인 필수**: 코드를 수정하기 전에 반드시 수정 계획을 보고하고 명시적 승인을 얻어야 한다.
- **근거 기반 답변**: 사용자의 질문에 답변할 때는 반드시 도구(Tool)로 현재 상태를 검증한 후 답변한다.
- **자가 점검 강제**: 도구 호출 전 4가지 핵심 질문(설명 여부, 승인 여부, 영향 분석, 의견 교환)에 스스로 답한다.
- **Anti-Rush**: AI가 임의로 단계의 완료를 정의하지 않는다. 사용자가 명시적으로 다음 단계를 지시할 때까지 현재 단계에서 충분히 논의한다.

## 2. 보안 및 데이터 안전
- **데이터 보존 우선**: 파괴적인 명령어 실행 전 반드시 백업을 수행하며, 데이터 유실을 방지한다.
- **환경 격리**: Local(Devnet)과 Production(Mainnet) 환경을 엄격히 분리 관리한다.
- **시크릿 보호**: `.env` 파일 및 프라이빗 키를 절대 커밋하지 않는다. `SOLANA_AGENT_PRIVATE_KEY` 등 온체인 키는 특히 주의한다.

## 3. 개발 표준 및 품질
- **표준 아키텍처 준수**: React Router v7 SSR, LangGraph, Solana web3.js 공식 패턴을 따르며 임시 방편을 금지한다.
- **영향 범위 분석**: 공통 로직(auth, db, ai) 수정 시 기존 기능에 영향이 없도록 격리하여 구현한다.
- **타입 안전성**: TypeScript 타입 에러를 허용하지 않는다. `~/*` 경로 별칭을 일관되게 사용한다.
- **DB 안전**: `npm run db:reset`은 파괴적 명령이다. 실행 전 반드시 사용자 확인을 받는다.

## 4. 문서 관리 표준 (5-Layer Structure)
개발 문서는 다음 5단계 구조를 엄격히 준수한다.

| 순번 | 폴더명 | 역할 |
| :--- | :--- | :--- |
| 1 | `docs/01_Concept_Design/` | 컨셉, 비전, UI 디자인, 기능 설명 |
| 2 | `docs/02_UI_Screens/` | UI 스크린, 화면 흐름, 프로토타입 리뷰 |
| 3 | `docs/03_Technical_Specs/` | DB 스키마, API 명세, 기술적 약속 |
| 4 | `docs/04_Logic_Progress/` | 백로그, 로드맵, 실행 계획, 비즈니스 로직 |
| 5 | `docs/05_QA_Validation/` | QA 체크리스트, 테스트 시나리오, 검증 |

## 5. Git 커밋 컨벤션
- **Conventional Commits**: `type(scope): 설명` 형식을 준수하며 한글로 상세 내용을 작성한다.
- **Type**: `feat`(기능), `fix`(버그), `docs`(문서), `style`(포맷), `refactor`(리팩토링), `test`(테스트), `chore`(기타)

## 6. 최종 약속
AI 에이전트는 본 **AGENTS.md**를 모든 판단의 최우선 근거로 삼는다. 문서에 정의되지 않은 작업을 수행할 경우 반드시 사용자에게 구현 전 확인을 먼저 구한다.
